Executive Summary

Sentinel is an autonomous Solana audit platform. This report covers the initial engagement period (2026-04-28) of continuous monitoring against Anatoly Yakovenko's Percolator perpetual DEX engine and BPF wrapper.

Sentinel runs as a persistent service on a hardened VPS. On every upstream commit, an autonomous hunt cycle fires: 100+ falsifiable invariant claims are dispatched as parallel agent reasoning tasks, each with iterative source-code-reading tools (read_file, grep, find_function). Every verdict is cited to specific file paths and line numbers and persisted into a SQLite findings database with an enforced lifecycle state machine.

This report summarises the verdicts of the initial deep-audit cycle (cycle 20260428-233639) and the surrounding speculation-mode and structural-class cycles run during the same engagement window.

Top-line conclusion

Across all hypotheses verified by the tool-using deep-audit layer, the engine's core conservation invariants are correctly enforced on the audited SHA. Specifically: SAFETY ATTESTATION is granted on the MAX_VAULT_TVL cap, the account-GC zeroing semantics, the position-zero basis-clear flow, the bedge accounting on partial closes, and the bounded square-root math. The single FALSE verdict (BY DESIGN) reflects the engine's intentional decoupling of insurance drains from vault accounting — which is the documented post-F7 design and is correctly enforced by the V >= C_tot + I module-level invariant.

The previously disclosed bugs #60 (WithdrawCollateral K-walk, qedbot) and #62 (funding-driven K-walk, dcccrypto) are independently verified as fixed on the audited SHA — every accrue-bearing wrapper handler routes through ensure_market_accrued_to_now_for_account_limited_op with the sole exception of KeeperCrank, which is intentionally paired with per-candidate touch_account_live_local (the e25601c fix referenced by issue #62).

Safety Attestations

The following invariants were verified under the audited SHA. Each verdict was rendered by an autonomous tool-using agent that iteratively explored the source code; line citations are reproduced verbatim below.

SNTNL-PERC-ATT-001

MAX_VAULT_TVL cap is enforced at every reachable mutation site

Safety Attestation Attested · HIGH confidence Hypothesis V4-vault-cap-respect
Affected files
engine/src/percolator.rs · engine/src/i128.rs
Mutation sites
L 5041, 5080, 6492, 6928, 7041, 7117, 7143, 7385
Backstop
L 4072–4076 — assert_public_postconditions_fast

Statement

For every reachable state transition, self.vault <= MAX_VAULT_TVL holds. All vault-increasing paths perform a checked_add followed by an explicit > MAX_VAULT_TVL rejection before committing the new value. All vault-decreasing paths use either checked_sub (which returns an explicit error on underflow) or a pre-checked subtraction guarded by an explicit comparison. The assert_public_postconditions_fast backstop independently re-verifies the bound at the end of every public instruction's exit path.

Evidence

Three vault-increase paths and their guards:

// engine/src/percolator.rs (deposit_not_atomic, L 5006–5013)
let v_candidate = self.vault.get().checked_add(amount).ok_or(RiskError::Overflow)?;
if v_candidate > MAX_VAULT_TVL { return Err(RiskError::Overflow); }
...
self.vault = U128::new(v_candidate);  // L 5041
// engine/src/percolator.rs (top_up_insurance_fund, L 7025–7031)
let new_vault = self.vault.get().checked_add(amount).ok_or(RiskError::Overflow)?;
if new_vault > MAX_VAULT_TVL { return Err(RiskError::Overflow); }
...
self.vault = U128::new(new_vault);  // L 7041
// engine/src/percolator.rs (deposit_fee_credits, L 7365–7371)
let new_vault = self.vault.get().checked_add(pay).ok_or(RiskError::Overflow)?;
if new_vault > MAX_VAULT_TVL { return Err(RiskError::Overflow); }
...
self.vault = U128::new(new_vault);  // L 7385

Backstop

// engine/src/percolator.rs (assert_public_postconditions_fast, L 4072–4076)
let vault = self.vault.get();
...
if vault > MAX_VAULT_TVL || capital > vault || insurance > vault {
    return Err(RiskError::CorruptState);
}

Edge cases considered

  • Initial state: vault begins at U128::ZERO (engine L 1336) — within the cap.
  • Saturating subtraction: U128::Sub saturates rather than panics. The two direct subtractions at L 6492 and L 6928 are pre-guarded by explicit comparisons (L 6489, L 6925) — so saturating semantics cannot produce an incorrect post-state.
  • Internal helpers (fee_debt_sweep, charge_fee_to_insurance, sweep_empty_market_surplus_to_insurance) redistribute balances within vault and never increase self.vault.

Tooling

26 tool-call turns. 44 distinct file reads / greps / function lookups. Cycle 20260428-233639. Full agent transcript in examples/V4-vault-cap-respect_response.md.

SNTNL-PERC-ATT-002

Account GC zeroes every byte before a slot is reused

Safety Attestation Attested · HIGH confidence Hypothesis AC1-account-gc-state-leak

free_slot / reclaim_empty_account zero every byte of the account slot before it is available for re-materialization. A subsequent materialize_at on the same index cannot inherit shadow state (PnL, capital, position basis) from the prior owner.

Verdict: TRUE / HIGH. 33 tool-call turns. Cycle 20260428-233639.

SNTNL-PERC-ATT-003

Square-root computations are bounded under the engine's own constants

Safety Attestation Attested · HIGH confidence Hypothesis AR6-square-root-bounds

Every sqrt-based computation (used in vega-style adjustments) is bounded by the engine's documented constants and never produces NaN-equivalents on integer arithmetic. 38 tool-call turns. Cycle 20260428-233639.

SNTNL-PERC-ATT-004

Position reaching zero atomically clears all basis-related fields

Safety Attestation Attested · HIGH confidence Hypothesis O7-position-zero-clears-basis

When position_q reaches exactly 0, basis-related fields are zeroed atomically; subsequent fills cannot inherit stale basis. 36 tool-call turns. Cycle 20260428-233639.

SNTNL-PERC-ATT-005

Bedge accounting on partial closes correctly apportions realized PnL

Safety Attestation Attested · HIGH confidence Hypothesis O9-position-bedge-correct

The basis-edge accounting on partial position closes correctly apportions realized PnL between the closed and remaining size. 36 tool-call turns. Cycle 20260428-233639.

SNTNL-PERC-ATT-VERIF-001

Public disclosures #60 and #62 are fixed on the audited SHA

Patch Verification Verified · HIGH confidence

Issues #60 (WithdrawCollateral K-walker, qedbot) and #62 (funding-driven K-walker, dcccrypto) belong to the strict-helper-asymmetry family. Both are confirmed fixed on the audited wrapper SHA.

Sentinel inventoried every call site of the two accrue helpers ensure_market_accrued_to_now_with_policy (weak) and ensure_market_accrued_to_now_for_account_limited_op (strict, calls reject_account_limited_market_progress):

// wrapper/src/percolator.rs — call-site map
ensure_market_accrued_to_now_for_account_limited_op  (STRICT) → L 5755, 6429, 7075, 7404, 8807
ensure_market_accrued_to_now_with_policy             (WEAK)   → L 5998 (only)

The single residual weak-helper call (L 5998) is inside KeeperCrank, which is gated by per-candidate touch_account_live_local (the e25601c fix referenced in issue #62). The K-walk axis on both oracle and funding is closed.
