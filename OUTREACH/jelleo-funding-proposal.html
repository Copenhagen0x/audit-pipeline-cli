02 · Methodology

Jelleo is the F7 discovery methodology, codified

Jelleo is not service-hours sold by the week. It is the disciplined methodology that produced F7, codified into autonomous infrastructure that runs continuously. Funding scales the methodology — and its inaugural deployment on Percolator — into a self-improving system.

The 4-step pipeline

01

Invariant catalog

Each protocol gets a tailored hypothesis library. Today: 125 hand-curated invariants across 10 surface families (vault accounting, residual conservation, position bookkeeping, settlement ordering, risk gates, liquidation paths, etc.). Every entry is a falsifiable claim with file/line targets and severity classification — not free-form prose.

02

Tool-using verification

Each hypothesis is dispatched to a Claude agent equipped with read_file, grep, and find_function. The agent reads the actual source, walks the code paths, and renders a line-cited verdict (TRUE / FALSE / NEEDS_LAYER_2). No speculation — every claim is anchored to specific file:line references in the engine.

03

Empirical confirmation

Verdicts that need empirical proof escalate to autonomous Rust integration test generation. The agent writes a custom cargo test against the engine's own test helpers, compiles, runs, and classifies as PASS or FAIL. Three of these tests are public in the repo's examples/confirmed-tests/ directory — autonomously generated, compile cleanly, pass against the real engine.

04

Sibling derivation

Every confirmed finding feeds back into the catalog. The platform extracts the structural attack pattern and auto-derives N sibling hypotheses targeting the same class through different code paths. F7 itself produced the 12-hypothesis strict_helper_class sibling library on disk today. The catalog compounds with every disclosure.

This pipeline is what produced F7. Funding scales the same pipeline to ceiling-depth across the entire Percolator engine + wrapper, with the depth compounding cycle-over-cycle (see Section 03).

03 · Learning loop

Jelleo is measurably better today than it was 6 cycles ago. The hypothesis library grows with every disclosed bug — each finding auto-extends the catalog with sibling-pattern hypotheses. Coverage isn't linear; it compounds.

04 · Architecture

Four pillars · the autonomous immune system

Jelleo is not an audit tool. It's the active-defense infrastructure layer for Solana DeFi — four interlocking pillars that together replace static-PDF audit reports with adaptive, autonomous, ecosystem-composable security. Each pillar ships in the 12-month engagement, demonstrated on Percolator as inaugural deployment. Each component exists in primitive form across various tools today; no platform combines all four into the integrated autonomous-defense loop described below.

P1

Counterfactual mainnet detection

For every transaction hitting the deployed protocol, a parallel counterfactual simulation runs: "what if attack pattern X applied here?" When the counterfactual state diverges meaningfully from actual state, the platform flags the transaction in real time — before the attack chain completes. This catches operational and state-mutation attacks that point-in-time audits structurally cannot detect. Drift's $270M durable-nonce drain on April 1 would likely have produced state divergence between actual and counterfactual on the first attack-chain transaction — a pattern this layer is designed to flag before the rest of the chain executed.

Ships via: mainnet RPC subscription pipeline · LiteSVM forked-state simulator · attack-pattern instrumentation library · state-divergence detector · real-time alert router. Existing primitives in repo: shadow (Layer 6 mainnet shadow audit).

P2

Cross-protocol bug-class propagation

When a bug is disclosed anywhere in the ecosystem (Immunefi, public PR, audit report, security advisory), Jelleo auto-extracts the structural attack pattern and instantly searches for the same class across every indexed protocol. Vulnerable instances are flagged within minutes — not at the next audit cycle months later. Every disclosure compounds the methodology across all monitored protocols. The library gets sharper with every bug found in the entire industry.

Ships via: disclosure-feed ingestion (Immunefi + GitHub security advisories + public PRs) · structural pattern extractor · cross-protocol search · vector DB for hypothesis embeddings. Existing primitives in repo: propagate, learn-from-disclosures.

P3

Closed-loop fix bundle

When a bug is confirmed, Jelleo doesn't stop at the finding. The platform generates a candidate fix, formally proves (via Kani harness) that the fix preserves all other invariants, validates against the full test suite, and bundles bug + fix + formal proof + tests into a single PR. Maintainers receive verified fix proposals — not raw findings — eliminating the cognitive load of designing the fix and proving it doesn't break anything else.

Ships via: fix-generation prompt loop · Kani harness synthesizer · full test-suite runner · PR-bundler. Existing primitives in repo: confirm, synth-kani, Ed25519 signing.

P4

On-chain attestation registry

Every Jelleo cycle produces a cryptographically-signed Merkle root attesting which invariants were verified at which commit SHA. The registry is published on-chain, queryable, and composable. Other protocols (lending oracles, insurance markets, bridges, custody platforms) can require Jelleo attestation as a precondition for protocol interaction — the security primitive becomes ecosystem-wide composable infrastructure, not a per-customer service. This is the layer that makes "verified continuous audit" something other Solana protocols can depend on cryptographically.

Ships via: Solana Anchor program (the registry contract) · Ed25519 signing pipeline · Merkle root indexer · public query API · composability documentation. Existing primitives in repo: Ed25519 signing, signed disclosure packages.

How the pillars compose

P1 detects in real time. P2 propagates defenses across protocols within minutes. P3 closes the loop from disclosure to verified fix without maintainer cognitive load. P4 makes every audit cycle cryptographically composable, queryable, and ecosystem-wide. Together they replace the static-PDF audit-report model with adaptive, autonomous, on-chain-composable security infrastructure. The 12-month engagement ships MVPs of all four pillars on Percolator. Year 2+ scales the model ecosystem-wide.
