Verify a cycle offline

Each cycle's merkle.json sidecar contains the leaves, the tree, and the signed root. Anyone can recompute the root from the leaves and check the signature against the platform pubkey — no Jelleo service required.

# 1. Fetch the cycle's merkle sidecar
curl -s https://api.jelleo.com/cycles/<root>.merkle.json > cycle.json

# 2. Recompute the root from leaves and verify the signature
audit-pipeline merkle verify --file cycle.json \
  --pubkey https://jelleo.com/keys/jelleo.ed25519.pub

What's inside a Merkle root

Each leaf is a SHA-256 hash of one finding's normalized fields — including finding_id, severity, protocol, poc_fired, engine_sha, and wrapper_sha. The schema is versioned (v2) so a future change to the leaf format can't silently mutate historical roots.
